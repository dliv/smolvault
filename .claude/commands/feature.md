## Instructions

1. Create a new feature document at `docs/features/$ARGUMENTS.md` using the template from `docs/features/_template.md`

2. Read the following context files:
   - `CLAUDE.md` - project overview
   - `docs/architecture.md` - system design

3. Work with the user to flesh out the feature document:
   - Clarify the goal
   - Define user stories
   - Sketch the API design
   - Identify data model changes
   - Break down into tasks

4. Stay in planning mode until the approach is clear and the user is ready to implement.
